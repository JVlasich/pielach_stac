# CLI
```
python opals2stac.py -inFile PATH [-outFile PATH] [-cfgFile PATH]
                     [--no-thumb] [--unknown-policy {skip,warn,register}] [-v]
```
später: `--init` und `--preset`
# Asset Discovery:
- Rekursiver Walk von `-inFile`.
- Stem-Pattern-Match gegen asset registry.
- Asset-Overrides aus `project.yml` werden gemerged.
- Unbekannte Files: Verhalten via `--unknown-policy`, default `warn`.
# Metadaten-Extraktion
- Selbsterklärend
# Item-Builder
- Gruppierung pro `(stem, category)`.
- `pystac.Item` mit Geometry (immer EPSG:4326), Datetime, Properties.
- Extensions
- Raster mit gleichem Stem wie eine Pointcloud erbt deren Datetime.
# Default-Hierarchie:
- Root-Catalog: id aus Verzeichnisname, title Title-Case, license `proprietary`.
- Sub-Kataloge dynamisch aus gefundenen Kategorien. Leere Zweige werden nicht erzeugt.
- Tree-Form analog zu `multimodal` [[Der Rote Faden#Was ist ein guter default für Kataloge falls nicht über ordnerstruktur festgelegt? |hier]]
- `temporal_extent` und `spatial_extent` einer Collection: Union der Item-Werte (indifferent, keine Zeit- oder Ort-Subdivision).
# Catalog-Manager:
- Wenn `outFile/catalog.json` existiert: Extend-Mode automatisch.
- Match auf `Item.id`. Vorhandenes Item: Assets mergen, identische hrefs ueberspringen.
- Geometry und Datetime nicht ueberschreiben.
- Collection-Extent nach Merge neu berechnet.
- Statische JSONs schreiben.
